<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → Firebase Auth | User credentials cross from device to Firebase Auth service |
| Client → Firestore | User data written to cloud database |
| Guest → Authenticated | Guest data migrated to authenticated user document |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-01 | Spoofing | AuthService | high | mitigate | Firebase Auth handles credential verification server-side; client never validates tokens |
| T-03-02 | Tampering | FirestoreService | medium | mitigate | Firestore security rules will enforce owner-only access (uid must match doc path); implemented in Firebase Console, not in code |
| T-03-03 | Info Disclosure | auth_provider.dart | low | accept | StreamProvider exposes User object to widget tree — standard Firebase pattern, no sensitive fields beyond uid/email |
| T-03-04 | Elevation of Privilege | Guest migration | medium | mitigate | Migration copies only onboarding prefs and local progress; does not grant elevated Firestore access beyond owner scope |
</threat_model>
