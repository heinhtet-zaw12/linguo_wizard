# Plan 01: Foundation & Core Services

## Objective

Build the complete foundation layer for Linguo Wizard: install all required dependencies, create core services (STT, TTS, AI), define data models, load curated scenario data, build the scenario selection screen, and wire splash-to-selection navigation.

## Tasks

### Task 1: Install Dependencies and Create Core Infrastructure

**Files:** pubspec.yaml, lib/core/config/app_config.dart, lib/core/services/stt_service.dart, lib/core/services/tts_service.dart, lib/core/services/ai_service.dart, lib/features/conversation/models/message.dart, lib/features/conversation/models/scenario.dart

**Read first:** pubspec.yaml, lib/core/theme/app_theme.dart, .planning/phases/01-foundation-core-voice-loop/01-RESEARCH.md

**Action:**

Step 1 — Add dependencies to pubspec.yaml under the existing `dependencies:` key (after `google_fonts`):
- `flutter_riverpod: ^2.6.0`
- `speech_to_text: ^6.8.0`
- `flutter_tts: ^4.0.0`
- `google_generative_ai: ^0.4.6`
- `shared_preferences: ^2.3.0`
- `device_info_plus: ^10.1.0`
- `uuid: ^4.5.0`
- `path_provider: ^2.1.0`

Then run `flutter pub get`.

Step 2 — Create `lib/core/config/app_config.dart`. Static class `AppConfig` with:
- `static const String geminiApiKey = String.fromEnvironment('API_KEY');`
- `static const String geminiModel = 'gemini-1.5-flash';`
- `static const int maxConversationTurns = 20;`
- `static const Duration sttListenTimeout = Duration(seconds: 30);`
- `static const Duration sttPauseTimeout = Duration(seconds: 3);`

Step 3 — Create `lib/core/services/stt_service.dart`. Class `SttService` wrapping `speech_to_text`:
- `Future<bool> initialize()` — calls `_speech.initialize()`
- `Future<void> startListening({required SpeechResultListener onResult, SpeechSoundLevelChange? onSoundLevel, String? localeId})` — uses `SpeechListenOptions(partialResults: true, cancelOnError: true, listenFor: AppConfig.sttListenTimeout, pauseFor: AppConfig.sttPauseTimeout)`
- `Future<void> stopListening()` — calls `_speech.stop()`
- `bool get isListening`

Step 4 — Create `lib/core/services/tts_service.dart`. Class `TtsService` wrapping `flutter_tts`:
- `Future<void> initialize()` — sets language "en-US", speech rate 0.5, volume 1.0, pitch 1.0, calls `awaitSpeakCompletion(true)`
- `Future<void> speak(String text)`
- `Future<void> stop()`
- `void setCompletionHandler(VoidCallback onComplete)`

Step 5 — Create `lib/core/services/ai_service.dart`. Class `AiService` wrapping `google_generative_ai`:
- `void initializePersona({required String personaName, required String personaDescription, required String scenarioGoal})` — creates `GenerativeModel` with `model: AppConfig.geminiModel`, `apiKey: AppConfig.geminiApiKey`, `systemInstruction` composing persona. System instruction must: stay in character, keep responses 1-3 sentences for spoken conversation, gently correct grammar naturally
- `Future<String> sendMessage(String userText)` — calls `_chat!.sendMessage(Content.text(userText))`, returns `response.text ?? ''`
- `bool get isReady`

Step 6 — Create `lib/features/conversation/models/message.dart`:
- Enum `MessageSender { user, ai }`
- Class `Message` with fields: `String id`, `MessageSender sender`, `String transcript`, `DateTime timestamp`, `Duration? audioDuration`
- Factory constructor `Message.create({required MessageSender sender, required String transcript})` auto-generates ID via uuid and timestamp

Step 7 — Create `lib/features/conversation/models/scenario.dart`:
- Class `Scenario` with fields: `String id`, `String title`, `String description`, `String personaName`, `String personaDescription`, `String goalDescription`, `String cefrLevel`, `String category`, `String openingMessage`
- `factory Scenario.fromJson(Map<String, dynamic> json)`

**Verify:** `flutter pub get && dart analyze lib/core/ lib/features/conversation/models/`

**Acceptance criteria:**
- pubspec.yaml contains all 7 new dependencies
- flutter pub get succeeds
- dart analyze reports 0 errors
- AppConfig, SttService, TtsService, AiService compile with correct signatures
- Message and Scenario models have all required fields

### Task 2: Create Scenario Data, Provider, and Selection Screen

**Files:** assets/data/scenarios/cafe_ordering.json, assets/data/scenarios/job_interview.json, assets/data/scenarios/airport_navigation.json, lib/features/scenario_selection/providers/scenario_provider.dart, lib/features/scenario_selection/screens/scenario_selection_screen.dart, lib/features/scenario_selection/widgets/scenario_card.dart

**Read first:** lib/features/conversation/models/scenario.dart, lib/core/theme/app_theme.dart, pubspec.yaml

**Action:**

Step 1 — Create 3 curated scenario JSON files under `assets/data/scenarios/`.

`cafe_ordering.json`: id="cafe_ordering", title="Ordering at a Cafe", category="travel", cefrLevel="A2", personaName="Emma", personaDescription="friendly barista at a cozy London cafe, patient with learners", goalDescription="Order a drink and a snack, ask about ingredients, handle payment", openingMessage="Hi there! Welcome to The Cozy Bean. What can I get for you today?"

`job_interview.json`: id="job_interview", title="Job Interview Practice", category="work", cefrLevel="B1", personaName="Mr. Collins", personaDescription="professional HR manager at a tech company, formal but supportive", goalDescription="Introduce yourself, describe your experience, answer behavioral questions", openingMessage="Good morning! Please, have a seat. I'm Mr. Collins from HR. Let's start with you telling me a bit about yourself."

`airport_navigation.json`: id="airport_navigation", title="Airport Navigation", category="travel", cefrLevel="A1", personaName="Alex", personaDescription="helpful airport information desk staff, speaks clearly and simply", goalDescription="Find your gate, ask about delays, request directions to the lounge", openingMessage="Hello! Welcome to Heathrow. How can I help you today?"

Step 2 — Add JSON assets to pubspec.yaml under `flutter: assets:` (after existing images):
- `- assets/data/scenarios/`

Step 3 — Create `lib/features/scenario_selection/providers/scenario_provider.dart`:
- `FutureProvider<List<Scenario>>` named `scenariosProvider` that loads all 3 JSON files via rootBundle, parses with jsonDecode, maps to Scenario.fromJson()
- `StateProvider<Scenario?>` named `selectedScenarioProvider`

Step 4 — Create `lib/features/scenario_selection/widgets/scenario_card.dart`:
- Stateless `ScenarioCard` with `Scenario scenario` and `VoidCallback onTap`
- Rounded corners (radius 16), pastel background, title (fredoka), description (quicksand), CEFR badge, category label
- Claymorphism box shadow

Step 5 — Create `lib/features/scenario_selection/screens/scenario_selection_screen.dart`:
- `ConsumerWidget` watching `scenariosProvider`
- `GridView.builder` with 2 columns, `ScenarioCard` for each
- Title "Choose a Scenario" (fredoka), gradient background
- On tap: set `selectedScenarioProvider`, navigate to `/conversation`

**Verify:** `dart analyze lib/features/scenario_selection/`

**Acceptance criteria:**
- 3 JSON scenario files with correct fields
- pubspec.yaml includes scenarios asset path
- scenariosProvider loads and parses all 3 scenarios
- ScenarioSelectionScreen shows grid of cards
- Tapping card navigates to /conversation
- dart analyze clean
