### Task 1: Conversation Provider with Voice Loop State Machine

**Files:** lib/features/conversation/providers/conversation_provider.dart

**Read first:** lib/core/services/stt_service.dart, lib/core/services/tts_service.dart, lib/core/services/ai_service.dart, lib/features/conversation/models/message.dart, lib/features/conversation/models/scenario.dart, lib/core/config/app_config.dart

**Action:**

Step 1 — Define `ConversationLoopState` enum:
```dart
enum ConversationLoopState { idle, recording, processing, speaking }
```

Step 2 — Define `ConversationState` class with fields:
- `List<Message> messages` (immutable list, default [])
- `bool isRecording` (default false)
- `bool isAiSpeaking` (default false)
- `String currentPartialTranscript` (default "")
- `ConversationLoopState loopState` (default IDLE)
- `Scenario? scenario` (default null)
- `int turnCount` (default 0)

Step 3 — Create `ConversationNotifier extends StateNotifier<ConversationState>`:
- Constructor initializes with default state
- `void initializeConversation(Scenario scenario)` — sets scenario, creates AI service persona using scenario.personaName, personaDescription, goalDescription, adds scenario.openingMessage as first Message with sender=ai
- `void startRecording()` — guard: only if loopState is IDLE; sets loopState=RECORDING, isRecording=true, clears currentPartialTranscript
- `void onPartialResult(String transcript)` — updates currentPartialTranscript
- `void stopRecording()` — sets loopState=PROCESSING, isRecording=false; passes final transcript to processUserMessage
- `Future<void> processUserMessage(String transcript)` — creates Message.create(sender: user, transcript), adds to messages, increments turnCount, calls AiService.sendMessage, creates Message.create(sender: ai, transcript: aiResponse), adds to messages, calls speakAiMessage
- `Future<void> speakAiMessage(String text)` — sets loopState=SPEAKING, isAiSpeaking=true, calls TtsService.speak(text), waits for completion, sets loopState=IDLE, isAiSpeaking=false
- Inject SttService, TtsService, AiService via constructor parameters
- Guard: if turnCount >= AppConfig.maxConversationTurns, do not allow new recordings

Step 4 — Expose as `conversationProvider` StateNotifierProvider

**Verify:** `dart analyze lib/features/conversation/providers/`

**Acceptance criteria:**
- ConversationLoopState enum has idle, recording, processing, speaking
- ConversationState has all fields with correct defaults
- State machine enforces: IDLE -> RECORDING -> PROCESSING -> SPEAKING -> IDLE
- Opening message added on initializeConversation
- dart analyze clean

### Task 2: Voice Message Bubble and Mic Button Widgets

**Files:** lib/features/conversation/widgets/voice_message_bubble.dart, lib/features/conversation/widgets/mic_button.dart

**Read first:** lib/features/conversation/models/message.dart, lib/features/conversation/providers/conversation_provider.dart, lib/core/theme/app_theme.dart

**Action:**

Step 1 — Create `lib/features/conversation/widgets/voice_message_bubble.dart`:

Class `VoiceMessageBubble extends StatelessWidget` with fields:
- `Message message`
- `bool isPlaying` (default false)

Design:
- **User messages**: right-aligned, primary pink background (AppColors.primaryPink), white text
- **AI messages**: left-aligned, white/light background with subtle shadow, dark text
- **Bubble shape**: rounded rectangle with border radius 16, asymmetric (less rounding on aligned side)
- **Transcript text**: below bubble in smaller font (quicksand, 13px, AppColors.textMuted)
- **Audio indicator**: mic icon for user, volume_up icon for AI inside bubble
- **Playing state**: when isPlaying=true and sender=ai, animated speaker icon
- **Claymorphism**: subtle box shadow matching AppColors.shadowPink

Step 2 — Create `lib/features/conversation/widgets/mic_button.dart`:

Class `MicButton extends StatelessWidget` with fields:
- `ConversationLoopState loopState`
- `VoidCallback? onPressed`

Design:
- Large circular button (72px) centered at bottom
- **IDLE**: solid primary pink, mic icon, white
- **RECORDING**: pulsing red/coral (AppColors.accentCoral), stop icon, animated ripple
- **PROCESSING**: loading spinner, button disabled (greyed out)
- **SPEAKING**: disabled, subtle grey, volume_up icon
- On tap: IDLE -> onPressed (start recording); RECORDING -> onPressed (stop recording); otherwise disabled

### Task 3: Conversation Screen and Navigation Wiring

**Files:** lib/features/conversation/screens/conversation_screen.dart, lib/main.dart

**Read first:** lib/features/conversation/providers/conversation_provider.dart, lib/features/conversation/widgets/voice_message_bubble.dart, lib/features/conversation/widgets/mic_button.dart, lib/features/scenario_selection/providers/scenario_provider.dart, lib/main.dart

**Action:**

Step 1 — Create `lib/features/conversation/screens/conversation_screen.dart`:

Class `ConversationScreen extends ConsumerWidget`:

Build method:
- Read `selectedScenarioProvider` to get current scenario
- Scaffold with AppColors.bgTop background
- SafeArea wrapping Column:
  - **Top bar**: Scenario title, goal text, back button
  - **Expanded**: ListView.builder of VoiceMessageBubble from state.messages, scroll to bottom on new message
  - **Partial transcript overlay**: when currentPartialTranscript non-empty and loopState is RECORDING, show subtle text above mic button
  - **Bottom**: MicButton with loopState and onPressed wired

State management:
- Use ConsumerStatefulWidget to manage service initialization
- In initState: create SttService, TtsService, AiService, initialize all, create ConversationNotifier, call initializeConversation(scenario)
- Set up STT result listener: partial -> onPartialResult; final -> stopRecording + processUserMessage
- Set up TTS completion handler: on complete, state returns to IDLE

Mic button handler `_onMicPressed()`:
- IDLE: startListening + startRecording
- RECORDING: stopListening + stopRecording
- Otherwise: do nothing

Step 2 — Modify `lib/main.dart`:
- Import ConversationScreen
- Replace placeholder '/conversation' route with `ConversationScreen()`
