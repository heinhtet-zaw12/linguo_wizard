## Typography

| Role | Size | Weight | Line Height | Font Family | Absorbed Former Roles |
|------|------|--------|-------------|-------------|----------------------|
| Display | 28px | 600 (semibold) | 1.2 | Fredoka | — |
| Heading | 20px | 600 (semibold) | 1.2 | Fredoka | Subheading (16px), Card title (18px) |
| Body | 14px | 500 (medium) | 1.5 | Quicksand | Card body (13px) |
| Label | 12px | 600 (semibold) | 1.4 | Quicksand | Caption (11px) |

**Notes:**
- Fredoka is used for all headings, titles, badges, and CTA text (playful, rounded aesthetic).
- Quicksand is used for body text, descriptions, subtitles, and UI labels (readable, neutral).
- Semantic role mapping: text previously styled as Subheading, Card title, or secondary heading uses `Heading 20px Fredoka 600`. Card body descriptions, subtitles, and secondary paragraphs use `Body 14px Quicksand 500`. Text previously styled as Caption or small secondary info uses `Label 12px Quicksand 600`.
- Letter-spacing 0.8px on uppercase label text (e.g., category labels on cards).

**Source:** Extracted from scenario_card.dart, home_screen.dart, scenario_selection_screen.dart, all existing widgets. Merged to 4 roles per checker constraint (Dimension 4).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | Pink gradient `#FADADD` -> `#F8C8D4` | Scaffold background, screen backgrounds |
| Secondary (30%) | White `#FFFFFF` | Cards, sheets, containers, chips (unselected) |
| Accent (10%) | `#F2A7B3` (primaryPink) | Active filter chips, primary buttons, mic states, interactive elements |
| Accent-dark | `#D4869A` (primaryPinkDark) | Links ("Browse All"), persona names, hover/pressed states |
| Accent-light | `#FCCDD6` (primaryPinkLight) | Chip borders (unselected), subtle highlights |
| Gold | `#F5C862` (accentGold) | Streak indicators, XP badges, achievement highlights, award elements |
| Coral | `#E8836B` (accentCoral) | Destructive actions only (delete, discard) |
| Text primary | `#3D2C35` (textDark) | Headings, card titles, primary content |
| Text muted | `#7A6570` (textMuted) | Descriptions, subtitles, empty state copy, secondary labels |
| Text light | `#FFFFFF` | Text on dark accent backgrounds (selected chips, buttons) |
| Shadow | `#30D4869A` (shadowPink) | Card shadows, container shadows (claymorphism depth) |

### Accent reserved for:

- **Primary Pink (#F2A7B3):** Active CEFR filter chips, Create Scenario FAB/action button, category tab selected indicator, scrollbar, loading indicator, search bar focus border
- **Gold (#F5C862):** Twist badge icon/award glow, Daily Challenge XP bonus indicator, streak flame active state, "2x XP" badge
- **Coral (#E8836B):** Delete custom scenario confirmation, discard unsaved scenario dialog, destructive button backgrounds only

**Never** use accent colors for: background fills (use dominant gradient), card surfaces (use white), body text (use textDark/textMuted), inactive elements (use white/primaryPinkLight).

## Screen-by-Screen Interaction Spec

### Screen 1: Scenario Selection (Redesigned)

**Location:** lib/features/scenario_selection/screens/scenario_selection_screen.dart

**Layout changes from Phase 1:**
1. **Category tabs row** — horizontal scrollable below the header, replacing the old single-row CEFR chips at the top:
   - 6 tabs: All, Travel, Work, Social, Academic, Daily Life
   - Selected tab: filled primaryPink, white text, rounded pill shape
   - Unselected tab: white fill, primaryPinkLight border, textDark text
   - Tabs use claymorphism-style icons next to label text (Material Icons: `travel_explore`, `work`, `people`, `school`, `home` — "All" uses `category`)
2. **CEFR filter chips** — remain as horizontal scrollable row BELOW category tabs (preserved from Phase 1)
   - All, A1, A2, B1, B2, C1 — same visual style as Phase 1
   - OR logic within selected category (show all scenarios matching category AND selected CEFR level)
3. **Search bar** — TextField between filter chips and grid:
   - Rounded container (borderRadius 20), white fill, primaryPink focus border
   - Search icon on left, clear (X) button on right when text is entered
   - Debounce: 300ms after user stops typing
   - Minimum characters: 2 before filtering triggers
   - Placeholder text: "Search scenarios..."
4. **Scenario grid** — 2-column GridView with scroll pagination:
   - `SliverGridDelegateWithFixedCrossAxisCount`, childAspectRatio: 0.78 (preserved)
   - 20 items loaded initially, 20 more on scroll to bottom
   - Loading indicator at bottom during pagination fetch (circular progress in primaryPink)
   - RefreshIndicator wrapper for pull-to-refresh
5. **"My Scenarios" section** — separate section after curated scenarios (for Plan 05-02):
   - Section header "My Scenarios" in Fredoka 20px weight 600
   - Horizontal scrollable row of custom scenario cards, or empty state message
6. **Create Scenario FAB** — bottom-right FloatingActionButton:
   - primaryPink background, white `add` icon
   - On tap: navigate to CreateScenarioScreen
7. **Twist badge** — on scenario cards that have been completed at least once:
   - Small badge overlay, top-right corner of card
   - Gold (#F5C862) sparkle/shuffle icon (Material: `auto_awesome` or `shuffle`)
   - Small "Twist" label or just the icon
   - On tap: starts conversation with twist variation instead of original
   - Twist variation passes through conversation pipeline unchanged

**Empty states:**
- **No scenarios loaded (error):** Error state copy with pull-to-refresh prompt
- **No scenarios match filters:** Empty state copy with hint to adjust filters
- **No custom scenarios:** Empty state copy with button to create one

### Screen 2: Create Scenario (New)

**Location:** lib/features/scenario_selection/screens/create_scenario_screen.dart

**Layout:**
1. **Header:** "Create a Scenario" (Fredoka 28px) + subtitle "Describe who you want to talk to" (Quicksand 14px muted)
2. **Form fields** (3 text inputs, vertically stacked with 16px gaps):
   - Field 1: "Who do you want to talk to?" — multiline TextField, hint text "e.g., a barista, a taxi driver"
   - Field 2: "Where are you?" — multiline TextField, hint text "e.g., at a busy coffee shop in London"
   - Field 3: "What's your goal?" — multiline TextField, hint text "e.g., order a flat white and ask about the menu"
   - All fields: white rounded container, primaryPink focus border, Fredoka 16px input text
3. **Generate button** — full-width primary button at bottom:
   - primaryPink background, white "Generate Scenario" text
   - On tap: calls AiService.generateScenario(), shows loading state
   - Loading state: "Generating your scenario..." with CircularProgressIndicator
4. **Preview card** — after generation completes, replace form with preview:
   - Reuse the ScenarioPreviewCard widget
   - Shows generated title, persona name, description, goal, opening message
   - "Try it" button (starts conversation) + "Discard" button (shows dialog)
5. **Save behavior:** Scenario is NOT saved to Firestore until user either starts the conversation or explicitly saves. On "Try it," save automatically to `users/{uid}/custom_scenarios/{id}` then navigate to conversation.

### Screen 3: Home Dashboard (Updated)

**Location:** lib/features/home/screens/home_screen.dart

**Addition: Daily Challenge hero card**
- Inserted between the streak/goal rings section and the "Recommended" section
- Visual style: white rounded container (borderRadius 20), full-width, prominent shadow
- Left edge: gold accent border (2px left border accentGold)
- Content rows:
  1. **Row 1:** "Today's Challenge" heading (Fredoka 18px weight 600) + gold "2x XP" badge (small pill, gold fill)
  2. **Row 2:** Challenge description (Quicksand 13px weight 500 muted, 2 lines max)
  3. **Row 3:** Countdown timer ("5h remaining") — updated every minute via Timer.periodic
     - If expired: "Ended — new challenge tomorrow" in muted text
  4. **Row 4:** "Start Challenge" button — full-width primaryPink rounded button, white text
     - On tap: navigate directly to conversation with the daily challenge scenario
- If daily challenge already completed today: show checkmark + "Challenge Complete! +100 XP" instead of button
  - Gold checkmark icon, green-ish success feel (use accentGold, not green — stays on-brand)

### Screen 4: Conversation Screen (Minimal Change)

**Location:** lib/features/conversation/ (unchanged structure)

- No visual changes to the conversation bubble design
- Daily Challenge scenario flows through same ConversationViewModel pipeline
- On conversation completion, HomeViewModel detects it was a Daily Challenge and applies 2x XP
- Twist scenarios are tracked via `isTwist` flag — same evaluation pipeline, same XP

### Visual Design Rules (Cross-Screen)

1. **Claymorphism card styling** (apply to all new containers):
   - White fill, borderRadius 16, dual box-shadow (pink shadow below + subtle white highlight above)
   - Pattern from existing scenario_card.dart
2. **All interactive elements** use primaryPink (#F2A7B3) as primary action color
3. **Gold (#F5C862)** reserved exclusively for achievements, XP, streaks, and awards — never for primary actions
4. **Coral (#E8836B)** is the ONLY destructive color — never use red
5. **Pull-to-refresh** on all scrollable data screens (scenario selection, home dashboard)
6. **Loading states:** Always use CircularProgressIndicator with primaryPink color, centered
7. **Primary visual focal point — Scenario Selection screen:** the 2-column scenario card grid. This is the dominant visual element users see first: rows of rounded white cards in a balanced masonry-like layout, each with a claymorphism shadow and an illustrative icon. The grid occupies roughly 70% of the viewport above the fold (after tabs, chips, and search bar).
8. **Secondary visual focal point — Scenario Selection screen:** the Create Scenario FAB (primaryPink circle, white `add` icon). Positioned bottom-right, it draws the eye as the only prominent floating element against the grid. Its pink color stands out from the white card field below.
