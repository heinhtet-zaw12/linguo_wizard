# Phase 5: Premium-and-Polish - Context

**Gathered:** 2026-07-23
**Status:** Ready for planning

<domain>
## Phase Boundary

This phase transforms the scenario system from 3 bundled JSONs to a production-scale Firestore-backed catalog (30+ curated scenarios, categories, search, pagination), enables users to create unlimited custom scenarios via AI generation, and adds daily engagement features (Today's Twist, Daily Challenge).

**Key decision: The app is completely free.** No premium gating, no subscriptions, no payment integration of any kind. Pronunciation-level scoring (paid APIs) is also deferred indefinitely.

Does NOT include: payment/subscription infrastructure, pronunciation scoring (Azure/Speechace), conversation history, vocabulary book, scenario collections, Myanmar UI, or in-conversation AI helper — those are Phase 6 or beyond.

</domain>

<decisions>
## Implementation Decisions

### Premium & Monetization (Removed)
- **D-01:** No premium tier. The app is completely free. No Stripe, no RevenueCat, no subscription flags, no payment UI.
- **D-02:** Plan 05-02 (custom scenarios) must be updated — remove "Free: 1/month, Premium: unlimited." All users get unlimited custom scenarios.

### UI Design (Locked)
- **D-14:** UI design is locked via `05-UI-SPEC.md` — screen layouts, copywriting, color tokens, typography, spacing, interaction behaviors. Downstream agents MUST read UI-SPEC before implementing.

### Today's Twist
- **D-03:** Twist is triggered via a badge on the scenario card. If a user has completed a scenario at least once, a "Twist" badge appears on that card. Tapping it starts the conversation with a variation.
- **D-04:** Progressive twist depth — first replay gets a subtle situational change (e.g., "Order coffee at a crowded cafe"), subsequent replays get a moderate change (different persona or goal).
- **D-05:** Twist is tracked as a visible badge on the card only. No detailed twist history or counter.

### Daily Challenge
- **D-06:** Each day, a unique AI-generated variation of a random curated scenario becomes "Today's Challenge." Fresh twist, generated daily.
- **D-07:** The Daily Challenge appears as a hero card on the Home dashboard — prominent placement for engagement.
- **D-08:** Completing the Daily Challenge awards 2x XP (50 base + 50 bonus = 100 XP total).

### Scenario Catalog
- **D-09:** The 30+ curated scenarios are generated by Gemini from a seed prompt, then reviewed before seeding into Firestore. AI-generated + human-reviewed workflow.

### Custom Scenario Lifecycle (New)
- **D-10:** Custom scenarios are fixed after creation — no editing. Users create a new scenario for variations.
- **D-11:** Users can delete saved custom scenarios from "My Scenarios" with a confirmation dialog ("This can't be undone").
- **D-12:** Custom scenarios in "My Scenarios" are ordered by creation date (newest first).
- **D-13:** No limit on custom scenario count — consistent with the app being completely free.

### Claude's Discretion
- Daily Challenge rotation logic: timezone handling, reset timing, global vs per-user, Cloud Function vs client-side
- Twist completion tracking: where/how replay count for progressive depth is stored (existing scenario completion data or new fields)
- Curated scenario schema: fields beyond the current Scenario model for the 30+ scenarios
- Scenario seed prompt design: Gemini prompt structure for generating quality scenarios
- Today's Twist Gemini prompt: how to generate subtle vs moderate variations
- Custom scenario deletion UX: specific interaction (long-press, swipe, or menu action) — planner's choice
- Search bar and empty state visual polish beyond what UI-SPEC defines
- Custom scenario generation output schema and validation

</decisions>

<specifics>
## Specific Ideas

- Today's Twist badge should feel playful — sparkle or shuffle icon (Material: `auto_awesome` or `shuffle`), not just text
- Daily Challenge hero card should show a countdown ("5h remaining") to create urgency, updated every minute
- Scenario category tabs should use claymorphism-style icons matching the app theme
- The "Create Scenario" FAB should be prominent — bottom-right, primaryPink, white add icon
- Gold (#F5C862) reserved for achievements, XP, streaks, awards — never for primary actions
- Coral (#E8836B) is the ONLY destructive color — never use Material red

</specifics>
