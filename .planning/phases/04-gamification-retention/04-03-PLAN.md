<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → Firestore (gamification writes from ConversationViewModel) | XP, streak, badge, SRS, and mistake data written after scenario completion; must be gated on successful evaluation |
| Client → Firestore (leaderboard reads) | Global leaderboard query; Firestore rules allow read access to users collection |
| UI → User (badge popup) | Badge popup is interruptive; must not block critical UI or cause accessibility issues |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-07 | Tampering | ConversationViewModel gamification trigger | high | mitigate | Gamification updates only fire after EvaluationService successfully returns ScoreData. If evaluation fails, no XP/streak/badge updates occur. |
| T-04-08 | Tampering | Leaderboard query manipulation | medium | mitigate | Firestore security rules enforce read access but leaderboard is public by design. XP inflation prevention is handled by T-04-04 (evaluation gate). |
| T-04-09 | Denial of Service | Badge popup blocking UI | low | accept | BadgePopup auto-dismisses after 4 seconds and is tappable. Not a security concern but UX consideration. |
| T-04-10 | Information Disclosure | Pre-scenario review SRS items | low | accept | SRS items are user's own practice data. Stored in user-scoped Firestore subcollection with security rules. |
| T-04-SC | Tampering | npm/pip/cargo installs | high | mitigate | No new package installs in this plan. confetti installed in Plan 01 (Approved). |
</threat_model>
