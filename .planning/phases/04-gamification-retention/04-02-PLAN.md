- `toJson()` and `fromJson()` factory for Firestore serialization

**lib/core/models/srs_item.dart** — SRS tracking (per D-13):
- `SrsItem` class with fields: id (String), text (String), category (String — 'vocabulary', 'grammar', 'phrase'), repetitions (int, default 0), easeFactor (double, default 2.5), interval (int, default 0), nextReview (DateTime), quality (int, default 0)
- `review(int quality)` method implementing SM-2: if quality < 3 => reset (repetitions=0, interval=1). If quality >= 3 => advance (first review interval=1, second interval=6, then interval = interval * easeFactor). Update easeFactor: EF = EF + (0.1 - (5-q) * (0.08 + (5-q) * 0.02)), min 1.3. Set nextReview = DateTime.now().add(Duration(days: interval)).
- `isDue` getter: DateTime.now().isAfter(nextReview)
- `toJson()` and `fromJson()` factory

**lib/core/models/mistake_record.dart** — Mistake tracking (per D-15, D-16):

- `toJson()` and `fromJson()` factory
- Designed to be stored in a Firestore subcollection users/{uid}/mistakes with 7-day rolling window

All models follow established patterns: PascalCase class names, camelCase fields, private constructor where appropriate, toJson/fromJson factories. Use `package:linguo_wizard/` imports for cross-feature references.

**lib/core/services/gamification_service.dart** — Coordinates gamification logic:
- `GamificationService` class (not abstract — concrete class for now)
- Constructor takes FirestoreService as dependency
- `Future<StreakData> updateStreak(String uid)` method:
  - Load current streak from Firestore (users/{uid}/progress document)
  - If no document exists, create with streak 1 and today's date
  - Calculate today as YYYY-MM-DD string in user's local timezone: `DateTime.now()` formatted as `YYYY-MM-DD`
  - Call streakData.updateForToday(today)
  - Save updated streak to Firestore
  - Return updated StreakData
- `int calculateLevel(int totalXp)` method: delegates to LevelConfig.getLevelInfo(totalXp). Returns level index.
- `List<Badge> checkBadges(String uid, {required int totalXp, required int currentStreak, required int scenariosCompleted, required ScoreData lastScore})` method:
  - Load already-earned badge IDs from Firestore (users/{uid}/badges subcollection)
  - For each BadgeDefinition in badgeDefinitions, check if condition is met and badge not already earned
  - Condition checks: 'streak' => currentStreak >= threshold, 'xp' => totalXp >= threshold, 'scenarios' => scenariosCompleted >= threshold, 'perfect_score' => lastScore.overallScore >= threshold, 'no_mistakes' => lastScore.grammarCorrections.isEmpty
  - For 'fast_learner': requires scenario duration (pass as optional parameter or skip for now — set threshold and document that duration check needs conversation start/end timestamps)
  - Return list of newly earned Badge objects
  - Save newly earned badges to Firestore async (fire-and-forget)
- `Future<void> awardXp(String uid, int xp)` method: increment totalXp in users/{uid}/progress document by xp. Per D-07: fixed 50 XP per scenario. Also increment scenariosCompleted.
- Per D-07: Use AppConfig.xpPerScenario (will be updated to 50 in Task 3) as the XP amount.

**lib/core/services/srs_service.dart** — SRS engine:
- `SrsService` class
- Constructor takes FirestoreService as dependency
- `Future<List<SrsItem>> getDueItems(String uid)` method: query users/{uid}/srs_items subcollection where nextReview <= DateTime.now(). Return list of SrsItem objects sorted by nextReview ascending.
- `Future<void> addItemsFromScore(String uid, ScoreData score)` method:
  - Extract grammar corrections from score.grammarCorrections and create SrsItem for each (category: 'grammar')
  - Extract vocabulary gaps (if score has vocabulary-related data) — if not available in ScoreData, create SrsItems from grammar corrections only for now
  - For each new item, check if an item with same text already exists in Firestore — if so, skip (don't duplicate)
  - Save new items to users/{uid}/srs_items subcollection
- `Future<void> reviewItem(String uid, SrsItem item, int quality)` method:
  - Call item.review(quality) to update SM-2 parameters
  - Save updated item back to Firestore
  - quality parameter: 0-5 scale where 0 = complete fail, 5 = perfect recall
- `Future<void> deleteItem(String uid, String itemId)` method: remove item from Firestore subcollection

Both services follow the established pattern: inject FirestoreService, use async methods, fire-and-forget for non-critical writes. Import models from core/models/ and config from core/config/.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → Firestore (gamification writes) | XP, streak, badge data written client-side; Firestore security rules must enforce owner-only access |
| Client → Firestore (SRS reads) | SRS items read per-user; must be scoped to authenticated user's subcollection |
| Client → Firestore (leaderboard reads) | Leaderboard reads are global but filtered by Firestore rules |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-03 | Tampering | Streak calculation | medium | mitigate | Streak stored in Firestore with server-side security rules; client calculates but Firestore is source of truth. Cloud Function validation can be added in Phase 5. |
| T-04-04 | Tampering | XP inflation | high | mitigate | XP awarded only after successful EvaluationService returns ScoreData. GamificationService.awardXp called from conversation completion flow, not from arbitrary client code. |

| T-04-06 | Information Disclosure | SRS data exposure | low | accept | SRS items are user-specific practice data, not sensitive. Stored in user-scoped subcollection with security rules. |
| T-04-SC | Tampering | npm/pip/cargo installs | high | mitigate | No new package installs in this plan. go_router and confetti installed in Plan 01 (both Approved). |
