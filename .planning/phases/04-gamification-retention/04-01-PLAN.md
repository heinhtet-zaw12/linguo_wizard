<objective>
Migrate the app from named routes in MaterialApp to GoRouter with a StatefulShellRoute.indexedStack for bottom navigation. This provides auth guards, deep linking, and stateful tab switching — the navigation foundation all Phase 4 features build on.

Purpose: Every screen in the app needs to live inside a navigation structure. Bottom nav is the primary UX pattern for multi-tab apps. GoRouter enables declarative route guards (auth check, onboarding check) and stateful tab preservation.

Output: GoRouter configured in main.dart with 4-tab bottom nav shell, route guards for auth and onboarding, and existing screens wired as routes. Guest users see 2 tabs, authenticated users see 4.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → GoRouter redirect | Route guard logic runs client-side; Firebase Auth state is the source of truth but redirect is bypassable |
| Guest → Auth tab visibility | Tab count is cosmetic only; route protection is in GoRouter redirect, not tab visibility |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-01 | Tampering | GoRouter redirect | medium | mitigate | Redirect checks FirebaseAuth.instance.currentUser (server-validated token); guest users cannot access /progress or /profile routes even if they manually navigate |
| T-04-02 | Information Disclosure | Tab visibility | low | accept | Guest tab hiding is UX, not security; all protected routes have server-side Firestore security rules |
| T-04-SC | Tampering | npm/pip/cargo installs | high | mitigate | go_router is official Flutter team package (Approved); confetti is established package (Approved). No [SUS] or [SLOP] packages. |
