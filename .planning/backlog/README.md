# 📋 Feature Backlog

Post-roadmap ideas that are captured but not yet committed. This is a **parking lot**, not a promise — entries get promoted to active phases when the time is right.

## How to add an idea

1. Create a new `.md` file with a short kebab-case name (e.g. `myanmar-ui.md`)
2. Use the template below
3. That's it — no plans, no estimates, no commitment

## Template

```markdown
---
status: captured
captured: YYYY-MM-DD
---

# Idea Title

**One-liner:** What's the gist?

**Why it matters:** User value, retention impact, or revenue angle.

**Rough scope:**
- Quick win (< 1 week)
- Medium feature (1-3 weeks)
- Major initiative (3+ weeks)

**Notes:** Any context, references, or early thoughts.
```

## Lifecycle

```
Captured  →  Triage (marked "next" or "later")
                ↓
         "next" → promoted to a Phase when current phase completes
         "later" → stays here, revisited periodically
```
